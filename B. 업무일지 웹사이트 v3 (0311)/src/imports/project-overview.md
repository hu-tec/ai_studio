[프로젝트 개요]

목표: 직원의 일일 업무와 AI 활용 내역을 시간대별로 기록하고, 관리자가 통합 조회할 수 있는 사내 어드민 웹사이트 프로토타입 구축.

기술 스택: React, Next.js, Vue 등 최신 프론트엔드 프레임워크와 UI 라이브러리(Tailwind CSS, MUI 등)를 자율적으로 선택하여 구현. 데이터는 시연용이므로 LocalStorage 또는 Mock 상태 관리(Zustand, Redux 등)로 구현할 것.

인증 상태: 별도 로그인 구현 생략. 항상 '로그인된 직원' 상태로 가정하며, 상단에 '관리자 리스트 보기'로 이동할 수 있는 버튼을 배치할 것.

[화면 1: 직원 업무 일지 작성 페이지]
1. 월간 캘린더 및 데이터 동기화 (핵심 기능)

화면 상단/좌측에 월간 캘린더 UI를 렌더링.

캘린더 내 직접 입력: 캘린더의 각 날짜 칸(Cell) 안에는 그날의 핵심 내용을 간단히 적을 수 있는 '기본 작성 칸(텍스트)'이 존재해야 함.

상세 펼침 및 데이터 연동: 특정 날짜를 클릭하면 캘린더 하단에 해당 날짜의 '상세 업무 일지 작성 영역'이 펼쳐짐. 캘린더 칸에 적은 내용과 하단 상세 영역의 '오늘 업무 요약' 데이터는 양방향으로 실시간 동기화(연동)되어야 함.

2. 다중 선택(Multi-Select) 분류 체계

다음 두 가지 항목은 반드시 다중 선택(Multiple Select) 체크박스 혹은 태그 방식으로 구현할 것.

홈페이지 분류 (대/중/소 3 Depth):

[대] 교육, 번역, 문서, 전시회, 전문가 매칭

[중] 아랍어 번역, 영상 번역, 웹툰 번역 (번역 하위) / 비즈 문서, 법률 문서, 의학 문서 (문서 하위) / 박물관, 전시회, 관광지 (전시회 하위)

소분류가 없는 경우 UI에서 비활성화 또는 생략.

부서 분류 (대/중/소 3 Depth):

[대] 기획, 홈피, 영업, 마케팅, 회계, 인사, 관리, 상담, 총무, 강사, 커리큘럼, 문제은행

현재 중/소분류 데이터가 없으므로 대분류만 다중 선택 가능하게 렌더링.

3. 시간대별 업무 일지 작성

입력 간격 선택: 30분 / 1시간 / 오전·오후 (Select Box 또는 Radio)

시간대별 행(Row) 자동 생성: 선택한 간격에 맞춰 업무 입력 행이 동적으로 주르륵 생성됨.

각 행의 구성: 시간대 | 제목 | 업무 내용 | 예정 사항 | [기본정보 및 AI 활용 작성] 버튼

4. 팝업/모달: 기본정보 및 AI 활용 상세 기록

동작 방식: 위 3번의 시간대별 행(Row)에 있는 **[기본정보 및 AI 활용 작성] 버튼**을 누르면, 해당 시간대의 업무에만 종속되는 팝업 창(Modal) 또는 확장 슬라이드가 열림. (하루 1개가 아니라, 시간대별 개별 작성임)

모달 내부 입력 항목:

작업 유형 (다중 체크박스): 문서/개발, 기획/업무 기획, 마케팅, 상담관리, 회계/총무, 전문가 관리, 강사커리, 기타

사용 AI (다중 체크박스): ChatGPT, Claude, Gemini, Copilot, Cursor, Midjourney, DALL-E, 기타

업무 지시 받은 구체적 사항 (Textarea):

텍스트 영역 내부에 "1. \n 2. \n 3. \n" 형태의 넘버링 텍스트가 기본값(Default Value)으로 미리 적혀 있어야 함.

하단에 별도의 [비고] 텍스트 입력 칸 제공.

프롬프트 기록 (Textarea):

[1차 프롬프트] 입력란

[2차 프롬프트] 입력란

결과물 (이미지 첨부 UI):

Before 이미지 파일 업로드 버튼 및 미리보기 영역

After 이미지 파일 업로드 버튼 및 미리보기 영역

보안 프롬프트 기록 (Textarea 조합):

[1차 보안 프롬프트], [2차 보안 프롬프트]

[문제점], [장점], [비고] (각각 개별 텍스트 입력칸으로 구성)

[화면 2: 관리자(CEO) 대시보드 리스트]
1. 페이지 목적

모든 직원의 날짜별, 시간대별 업무 일지와 AI 활용 내역을 한눈에 조회하고 필터링하는 통합 리스트.

2. 레이아웃 및 기능

상단 다중 필터: 기간(Date Picker), 부서(다중 선택 필터), 홈페이지 분류, 작업 유형, 사용 AI별로 필터링 가능하게 구현.

데이터 테이블:

컬럼: 날짜 | 부서 | 작성자 | 시간대 | 업무 제목 | 사용 AI(배지 형태) | 상세보기

상세보기 모달: 목록에서 '상세보기' 클릭 시 해당 직원이 그 시간대에 작성한 프롬프트 내용, 이미지(Before/After), 보안 프롬프트 등의 모든 모달 데이터를 읽기 전용(Read-only)으로 표시.

[개발 필수 가이드]

시연(Prototype) 목적이므로 백엔드 API 연동을 기다리지 말고, Mock Data(더미 데이터)를 최소 5일 치, 여러 명의 직원 분량으로 풍부하게 생성하여 앱 초기 로드 시 반영할 것.

복잡한 폼(Form)이 많으므로 react-hook-form 같은 라이브러리를 사용하여 상태 렌더링 최적화를 고려할 것.
