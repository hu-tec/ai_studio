사내 업무지침 DB를 관리하는 데스크톱 웹 대시보드를 디자인해 줘.

[디자인 스타일 및 레이아웃 규칙]



스타일: 100% 엄격한 모노톤 (검정, 흰색, 회색 계조만 허용). 포인트 컬러 금지. 깔끔한 와이어프레임 스타일.

레이아웃: 화면 전체 너비를 꽉 채우는 Full-width 구조. 스크롤이 길어지더라도 모든 정보가 한눈에 보이도록 정보 밀도를 높게 설정.

초기 상태: 모든 카테고리와 데이터가 접힘 없이 100% 펼쳐진 상태(All Expanded)로 디자인.

[전체 구조 설계: 3개의 메인 섹션]

대시보드 본문은 아래로 스크롤하며 볼 수 있도록 3개의 큰 세로 구역(회사 전체, 부서별, 직급별)으로 나눔.

1. 회사 전체 (Company-wide) 섹션



섹션 타이틀: "회사 전체 공통 지침" (크고 굵은 텍스트) + 우측에 텍스트 버튼 [🔽 접기]

내부 레이아웃: 3개의 열(3-Columns)로 동일하게 분할.

1열 헤더: [규정] 고정 (변경금지) / 짙은 회색 배경, 흰 글씨

2열 헤더: [준규정] 준고정 (조건부변경가능) / 중간 회색 배경, 검은 글씨

3열 헤더: [선택사항] 선택고정 (언제든변경가능) / 밝은 회색 배경, 뚜렷한 테두리

각 열 아래에 텍스트 형태의 세부 지침 리스트를 나열. 각 리스트 우측 끝에 [+추가] [삭제] 텍스트 버튼 배치.

2. 부서별 (By Department) 섹션



섹션 타이틀: "부서별 업무 지침" + 우측에 텍스트 버튼 [🔽 접기]

내부에 '기획', '영업', '개발' 등 부서명을 나타내는 서브 타이틀 블록들을 세로로 나열.

각 부서명 블록 바로 아래에 위 1번과 동일한 [규정] | [준규정] | [선택사항] 3열 그리드를 각각 삽입하여 부서별 규칙을 보여줌.

3. 직급별 (By Rank) 섹션



섹션 타이틀: "직급별 업무 지침" + 우측에 텍스트 버튼 [🔽 접기]

내부에 '신입', '팀장', '임원' 등 직급명을 나타내는 서브 타이틀 블록들을 세로로 나열.

각 직급명 블록 바로 아래에도 위와 동일한 [규정] | [준규정] | [선택사항] 3열 그리드를 각각 삽입.

[공통 컴포넌트 규칙]



모든 섹션과 카테고리 블록은 시각적으로 구분되도록 얇은 회색 테두리(Stroke) 사용.

버튼은 복잡한 아이콘 대신 텍스트(예: [+추가]) 형식을 사용하여 MVP의 목적에 맞게 직관성 극대화.
