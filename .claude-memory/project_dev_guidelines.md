# 휴텍씨 개발 규정교육 (전문)

## ★ 핵심 원칙 (절대 잊지 말 것)
- **'한 눈에 모든 게 담기게!'** 가 최우선
- 설계: 처음부터 디테일하게
- 기능 구현: 최대한 기능 다 붙이고, 피드백 받아서 '삭제'
- 목표 설정: 항상 구체적인 목표를 명시
- 주요사항: 항상 메인(주인공)과 서브(조연)를 알려주기
- 금지사항: 금지할 사항들 명시 필수
- 규정 설정: 항상 모델에게 규정들을 먼저 학습시키기
- 주제별로 모아서 숫자로 나열
- 보고할 때는 표로 '한 눈에' 볼 수 있게

## 1. 업무 설계의 기본 원칙

### 1-1. 프레임 우선 설계
- 공통 틀 우선 제작: 먼저 하나의 공통 틀을 만들고 이후 여러 업무에 반복 적용
- 찍어내기 구조 지향: 틀 완성 후 커리큘럼, 교재, 시험, 랜딩페이지 등 확장 생산
- 매뉴얼 선행 정리: 시간 부족할수록 매뉴얼과 규정부터 정리하는 것이 우선

### 1-2. 구조 설계 기준
- 확장성 중시: 레이아웃은 다른 업무로 쉽게 확장 가능해야 함
- 공통과 개별 분리: 전 부서 공통 항목과 부서별 개별 항목 분리 설계
- 번호 체계 부여: 3-1, 3-2 처럼 번호 붙여 버전과 위치 구분

## 2. 분류 기준 및 규정 체계

### 2-1. 기본 분류 축
- 대·중·소 분류: 업무와 자료는 대-중-소 단계적 구조
- 급수 구분: 시험, 교육, 콘텐츠는 급수 기준과 연결
- 분야별 구분: 분야 따라 필요 항목과 구조가 다름

### 2-2. 규정 구분
- **고정 규정**: 반드시 유지되어야 하는 핵심 항목
- **준고정 규정**: 기본 구조 유지하되 업무 성격에 따라 일부 변형 가능
- **선택 규정**: 필요시에만 추가/제외 가능한 가변 항목

### 2-3. 분야별 차이 반영
- 법률, 고전 번역, 마케팅 등 분야마다 필수 항목 다름
- 공통 틀 위에 각 분야 세부 입력 항목 덧붙이기

## 3. 적용 대상 업무

### 3-1. 교육 및 시험 제작
- 커리큘럼 설계, 교재 제작, 문제은행 구축

### 3-2. 홈페이지 및 랜딩페이지 제작
- 랜딩페이지 설계 (연도별, 주제별, 뉴스형)
- 히어로 섹션 정의
- 홈페이지별 독립 구성 (TESOL, 번역, 시험 등)

### 3-3. 마케팅 제작
- 헤드라인, 핵심 키워드, 대상자 설정 등 별도 구조
- 타깃 분류 (전문가 여부, 연령대, 학습 상태 등)

## 4. 화면 및 기능 설계

### 4-1. 입력 및 선택 기능
- 체크박스: 고정/준고정/선택 구조를 빠르게 조합
- 활성/비활성 토글
- 텍스트 입력 박스
- 버튼 바: 유형 빠르게 전환

### 4-2. 보기 및 관리 기능
- **대시보드 보기**: 전체 현황 요약 화면
- **리스트 보기**: 한 줄씩 펼쳐 보기
- **미리보기 기능**: 작성 전후 미리보기
- **작성·추가·수정 기능**
- **필터 기능**: 특정 사람/조건 필터

### 4-3. 레이아웃 유형
- 장문형 / 질문·단답형 / 4지선다형 / 단일 선택형

### 4-4. 탐색 및 확장 기능
- 펼치기 기능 선호
- 대·중·소 확장 기능 (대분류→중분류→소분류)
- 한 화면 모드 전환 (대시보드형↔리스트형)

## 5. 작성 및 운영 방식

### 5-1. 입력 흐름
- 1차 선택 후 작성: 항목 체크 → 펼쳐져서 입력
- 엑셀 누적 정리 → 새 구조 안으로 이전
- 요약 대시보드화

### 5-2. 팀별 활용
- 가이드라인 배포: 각 팀에 동일 구조 가이드 제공
- 회사 방침 내재화: 면접, 교육, 제작물에도 고정/준고정/선택 규정 활용

### 5-3. 프롬프트 관리
- 프롬프트 기록화: 1차 결과, 2차 수정 결과 기록
- 재사용 가능한 노하우화: 개인 감이 아닌 회사 업무 노하우로

## 6. UI/UX 선호 방향

### 6-1. 사용성 중심 설계
- 좁고 빡센 틀 지양
- **한 줄형 선호**: 한 줄씩 펼쳐서 바로 쓰는 구조
- 과한 화려함 지양: 실용적으로
- 상태 가시성 강조: 고정/준고정 상태가 바로 보이게

### 6-2. 배치 원칙
- 페이지 분리 허용: 한 화면에 못 담으면 분리하되 연결 유지
- 3단·4열 활용

## 7. 홈페이지 구성 원칙

### 7-1. 기본 목차 원칙
- 서비스 소개 우선 (회사 소개는 후순위)
- 필수 항목: 실적, 전문가 신청, 견적 의뢰

### 7-2. 고전 번역 사이트 제작 원칙
- 기존 틀 재활용 → 고전용 변형
- 독립 사이트 운영 (하위가 아닌 별도 구조)

### 7-3. 제작 절차
- 자료 조사 선행 → 표 정리 → 목차 설계 → 누락 보완
- 내용 자료 + 스타일 참고 자료 함께 활용

**Why:** 이 규정은 휴텍씨의 모든 개발/디자인 작업의 기본 원칙. 대시보드, 화면, 문서 등 제작 시 항상 이 규정을 따라야 함.
**How to apply:** 화면 설계 시 대·중·소 분류 구조, 고정/준고정/선택 체계, 한 눈에 보기 원칙 적용. 기능은 최대한 넣고 피드백 후 삭제 방식.
